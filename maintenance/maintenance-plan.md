# Software Maintenance Plan

## Pendahuluan

Software Maintenance merupakan proses pemeliharaan perangkat lunak setelah sistem selesai dikembangkan dan digunakan oleh pengguna. Tujuannya adalah memastikan sistem tetap berjalan dengan baik, aman, serta mampu beradaptasi terhadap perubahan kebutuhan bisnis maupun teknologi.

Studi kasus pada dokumen ini adalah Sistem E-Commerce.

---

# Corrective Maintenance

### 1. Perbaikan Bug Login

**Deskripsi**

Beberapa pengguna melaporkan tidak dapat login meskipun username dan password sudah benar.

**Penyebab**

- Session tidak dibuat dengan benar.
- Validasi password gagal.

**Solusi**

- Memperbaiki proses autentikasi.
- Menambahkan logging untuk mendeteksi penyebab kegagalan.
